Vector Autoregressions

Vector Autoregressions (VAR) are probably the most commonly used method to model dynamics between multiple variables.

Vector Autoregressions (VAR)

A Vector Autoregression (VAR) is a simple model that links the current data to past observations and an error term. More formally, if we observe a set of variales, a vector of variables, every period and denote this vector with \(y_t\), then a VAR(1) model has the form \[ \mathbf{y}_t = \mathbf{c}+ \mathbf{B}_1\mathbf{y}_{t-1} + \mathbf{u}_t, \] where \(\mathbf{c}\) is some vector of constants, \(\mathbf{B}_1\) the the autoregressive (AR) matrix and \(\mathbf{u}_t\) is some error term with mean \(\mathbf{0}\) and covariance \(\boldsymbol{\Sigma}_u\). We call this model a VAR(1) model because it relates the current observation \(\mathbf{y}_t\) to a single lagged observation \(\mathbf{y}_{t-1}\).

The VAR(1) model can be generalised by extending the number of lags in the model. With \(p\) lags in the model, the VAR(p) â€“ note the \(p\) in parentheses â€“ becomes \[ \mathbf{y}_t = \mathbf{c}+ \sum_{i=1}^p \mathbf{B}_i \mathbf{y}_{t-i} + \mathbf{u}_t, \] where \(\mathbf{B}_i\) is the i-th autoregressive (AR) matrix.

So what does a VAR model do?

Above are the mathematical exppressions, but what does a VAR model actually do and why do people use it? These are good questions, and VAR models have been used in various ways and thus have also been motivated in various ways. The easiest motivation is to note that a VAR model simply links the current observation to past observation. As such, a VAR model is claiming that there are some dynamics in the system â€“ whether thatâ€™s an economy, a business, the oxygen level of a person, etc â€“ and that these dynamics are well described by a linear function that links the current observation to a finite past.

Naturally, there are many siutations in which both the linear and finite past aspects are doubtful. However, both concerns can somewhat be remedied. First, the finite past limitation can be lifted by working with VAR(\(\infty\)) models. Second, the linearity limitation cannot be lifted but can at least be defended in the following way.

Suppose the real system actually features non-linear dynamics â€“ however still with a finite past. We could express this mathematically as \[ \mathbf{y}_t = f(\mathbf{y}_{t-1}, \ldots, \mathbf{y}_{t-p}) + \mathbf{u}_t. \] Also suppose that this non-linear system has some fixed point \(\bar{\mathbf{y}}\) such that \[ \bar{\mathbf{y}} = f(\bar{\mathbf{y}}, \ldots, \bar{\mathbf{y}}) + \mathbf{0}. \] Thus, if all previous values are at the fixed point and the innovation (the error term) are zero, then the system will remain at the fixed point. If these two assumptions describe the true systemâ€™s dynamics, then the VAR(p) model can be considered as a linear approximation of the possibly complicated non-linear dynamics. Here, the approximation is taken around the fixed point. Thus, taking a first-order Taylor approximation around the fixed point we get \[ \begin{split} \mathbf{y}_t &= f(\bar{\mathbf{y}}, \ldots, \bar{\mathbf{y}}) + \sum_{i=1}^p\frac{\partial f}{\partial \mathbf{y}_{t-i}}(\mathbf{y}_{t-i} - \bar{\mathbf{y}}) + \mathbf{u}_t \\ &= \underbrace{\left[f(\bar{\mathbf{y}}, \ldots, \bar{\mathbf{y}}) - \sum_{i=1}^p\frac{\partial f}{\partial \mathbf{y}_{t-i}}\bar{\mathbf{y}}\right]}_{\mathbf{c}} + \sum_{i=1}^p \underbrace{\frac{\partial f}{\partial \mathbf{y}_{t-i}}}_{\mathbf{B}_i}\mathbf{y}_{t-i} + \mathbf{u}_t, \end{split} \] which is a VAR(p) model.

Other Forms

At times it can be convenient to represent the VAR(p) model in other, yet still equivalent, mathematical forms. Each form has advantages and disadvantages.

Matrix Form

The matrix form is best used when deriving computations methods. Data is naturally represented as matrices and thus having a matrix form allows us to link the VAR most naturally to data structures used by computers.

To move from the standard representation above to the matrix representation, we first take the transpose \[ \mathbf{y}_t' = \mathbf{c}' + \sum_{i=1}^p \mathbf{y}_{t-i}'\mathbf{B}_i' + \mathbf{u}_t'. \]

We next summarise this transposed system by introducing \(\mathbf{z}_t\) and \(\mathbf{B}_+\), where the former summarises all the data past data up to a finite horizon, and the latter collects all coefficients in a single matrix. \[ \mathbf{z}_t = \left[\mathbf{y}_{t-1}', \ldots, \mathbf{y}_{t-p}' \right], \quad \mathbf{B}_+ = \left[\mathbf{c}, \mathbf{B}_1, \ldots, \mathbf{B}_p \right] \]

Using \(\mathbf{z}_t\) and \(\mathbf{B}_+\), we can express the VAR(p) in the following way \[ \mathbf{y}_t' = \mathbf{z}_t\mathbf{B}_+' + \mathbf{u}_t'. \]

Moving from this form to the matrix form simply consists of stacking the equations for each time period underneath each other. Note though that the first equation is for \(t=p+1\), since no data exists for periods \(t\leq 0\). \[ \mathbf{Y}= \begin{bmatrix}\mathbf{y}_{p+1}' \\ \mathbf{y}_{p+2}' \\ \vdots \\ \mathbf{y}_{T-1}' \\ \mathbf{y}_{T}' \end{bmatrix}, \quad \mathbf{U}= \begin{bmatrix}\mathbf{u}_{p+1}' \\ \mathbf{u}_{p+2}' \\ \vdots \\ \mathbf{u}_{T-1}' \\ \mathbf{u}_{T}' \end{bmatrix}, \quad \mathbf{Z}= \begin{bmatrix}\mathbf{z}_{p+1} \\ \mathbf{z}_{p+1} \\ \vdots \\ \mathbf{z}_{T-1} \\ \mathbf{z}_{T} \end{bmatrix} \]

The final matrix form is then given by the following equation \[ \mathbf{Y}= \mathbf{Z}\mathbf{B}_+' + \mathbf{U}. \] Note that this is simply a matrix equation with unknown coefficients. We may thus attempt to solve for the unknowns by using least squares. However, since matrix differentation can be a bit tricky at times, and vector differentation is much more common, we may prefer thinking about the model in a vectorised rather than matrix representation.

Vectorised Form

To move from the matrix representation to the vectorised representation we apply the \(\text{vec}\) operator.1 \[ \begin{split} \mathbf{y}&= \text{vec}(\mathbf{Y}) \\ &= \text{vec}(\mathbf{Z}\mathbf{B}_+') + \text{vec}(\mathbf{U}) \\ &= \text{vec}(\mathbf{Z}\mathbf{B}_+'\mathbf{I}) + \text{vec}(\mathbf{U}) \\ &= (\mathbf{I}\otimes \mathbf{Z})\text{vec}(\mathbf{B}_+') + \text{vec}(\mathbf{U}) &= \mathbf{X}\mathbf{b}+ \mathbf{u} \end{split} \] Note that the last lines substitute \(\mathbf{X}= (\mathbf{I}\otimes \mathbf{Z})\), \(\mathbf{b}=\text{vec}(\mathbf{B}_+)\), and \(\mathbf{u}= \text{vec}(\mathbf{U})\) to more clearly show the connection to standard linear regressions. This system can now be solves using ordinary least squares or any other method, including penalised methods.

Companion Form

While the matrix form and the vector form are often used to derive computational methods to estimate a VAR(p), the companion form is more often used to analyse the dynamics of a VAR(p). Analysing dynamics of a VAR(1) is much easier than those of a VAR(p), since all we need to keep track of is last periodâ€™s value. However, it turns out that the previous statement is not entirely correct, the reason for which is the companion form. The companion form of a VAR(p) is basically a re-interpretation that shows that any VAR(p) can be written as a VAR(1) if we extend out state vector. State vectors do not usually come up in VAR analysis, however, \(\mathbf{y}_t\) is nothing but a vector of states, all of which observed, which evolves over time. The choice of states to include at time \(t\) is arbitrary, as long as no future information is included â€“ the basic assumption is that time \(t\) state vectors are fully observable at time \(t\).

We can exploit the state interpretation of \(\mathbf{y}_t\) and include past observations into the state vector. Thus, define \[ \tilde{\mathbf{y}}_t' = (\mathbf{y}_t', \mathbf{y}_{t-1}', \ldots, \mathbf{y}_{t-p+2}', \mathbf{y}_{t-p+1}'). \]

The initial VAR(p) relationship then implies the following dynamic relationship for \(\tilde{\mathbf{y}}_t\) \[ \tilde{\mathbf{y}}_t = \begin{bmatrix} \mathbf{B}_1 & \mathbf{B}_2 & \ldots & \mathbf{B}_{p-1} & \mathbf{B}_p \\ \mathbf{I}& \mathbf{O}& \ldots & \mathbf{O}& \mathbf{O}\\ \mathbf{O}& \mathbf{I}& \ldots & \mathbf{O}& \mathbf{O}\\ \vdots & \vdots & \ddots & \vdots & \vdots \\ \mathbf{O}& \mathbf{O}& \ldots & \mathbf{I}& \mathbf{O} \end{bmatrix} \tilde{\mathbf{y}}_{t-1} + \begin{bmatrix}\mathbf{u}_t \\ \mathbf{0} \\ \vdots \\ \mathbf{0} \\ \mathbf{0} \end{bmatrix}, \] which is a VAR(1) in the state vector \(\tilde{\mathbf{y}}_t\).

To move from the companion form back to the original state vector \(\mathbf{y}_t\), we simply define \[ \mathbf{J}= [\mathbf{I}, \mathbf{O}, \ldots, \mathbf{O}, \mathbf{O}] \] and pre-multiply \(\tilde{\mathbf{y}}_t\) by \(\mathbf{J}\) \[ \mathbf{y}_t = \mathbf{J}\tilde{\mathbf{y}}_t. \]

Thus, if we are able to analyse the dynamics of a VAR(1), then we are also able to analyse the dyanmics of a VAR(p). For that reason, much of the other sections will always first focus on the VAR(1) case and will then exploit the companion matrix to generalise the findings to a VAR(p) case.

KeynesVAR

KeynesVAR uses the matrix representation internally. As such, a VAR DGP is specified by providing \(\mathbf{B}_+\) and the covariance matrix for the error term \(\mathbf{u}_t\).

using LinearAlgebra
using KeynesVAR
using Random

K = 3  # Number of variables in the VAR
p = 4  # Number of lags

B = randn(K, K*p)  # No constant included
Sigma_u = diagm(ones(K))  # Covariance of error terms
dgp_var = VAR(B, Sigma_u)  # Specification of VAR DGP

B = randn(K, K*p + p)  # Including a constant
Sigma_u = diagm(ones(K))  # Covariance of error terms
dgp_var = VAR(B, Sigma_u)  # Specification of VAR DGP
